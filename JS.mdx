# JavaScript
웹 페이지에 동적인 기능을 추가하는 프로그래밍 언어
HTML과 CSS와 함께 웹의 핵심 기술 중 하나
클라이언트와 서버 측 모두에서 사용될 수 있다

# JavaScript와 ECMAScript의 차이점
ECMAScript는 JavaScript의 표준 사양을 정의한 것
JavaScript는 ECMAScript 사양을 구현한 언어
ES6 이후의 버전들은 ECMAScript의 최신 표준을 기반

# var, let, const의 차이점
var는 함수 스코프를 가지며 호이스팅
let과 const는 블록 스코프를 가지며, const는 재할당이 불가능
let과 const는 ES6에서 도입되어 더 안전한 변수 선언을 제공

# 클로저(Closure)
함수가 선언될 때의 렉시컬 환경을 기억하는 기능
내부 함수가 외부 함수의 변수에 접근할 수 있게 한다.
데이터 은닉과 함수형 프로그래밍에 유용

# 프로토타입(Prototype)
객체 지향 프로그래밍에서 상속을 구현하는 방법
모든 객체는 프로토타입을 가지고 있으며, 프로토타입 체인을 통해 상속된 속성과 메서드에 접근

# 비동기 처리(Asynchronous Processing)와 콜백(Callback)
비동기 처리는 코드가 블로킹 없이 실행
콜백은 비동기 작업이 완료된 후 실행되는 함수
JavaScript는 이벤트 루프를 통해 비동기 처리를 관리

# 프로미스(Promise)와 async/await의 차이점
프로미스는 비동기 작업의 완료 또는 실패를 나타내는 객체
async/await는 프로미스를 더 간결하고 직관적으로 사용할 수 있게 한다
await는 프로미스가 해결될 때까지 기다린다.

# JavaScript에서 this 키워드의 의미
this는 현재 실행 중인 함수나 객체를 참조 
함수 호출 방식에 따라 this의 값이 달라짐. 
전역 컨텍스트에서는 전역 객체를, 메서드 내부에서는 해당 객체를 가리킴.

# 이벤트 버블링(Event Bubbling)과 이벤트 캡처(Event Capturing)
이벤트 버블링은 이벤트가 하위 요소에서 상위 요소로 전파되는 방식 
이벤트 캡처는 상위 요소에서 하위 요소로 전파 
addEventListener 메서드의 세 번째 인자로 캡처링을 설정할 수 있다

# 화살표 함수(Arrow Function)와 일반 함수의 차이점
화살표 함수는 function 키워드 대신 =>를 사용하여 더 간결하게 작성된다. 
화살표 함수는 this 바인딩을 어휘적으로 상속받는다. 
생성자 함수로 사용할 수 없다.
